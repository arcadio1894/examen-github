  1. Que es un sistema de control de versiones?

    Es un sistema que registra y permite controlar los cambios (de manera independiente o grupal) realizados sobre un archivo o conjunto de archivos a lo largo del tiempo de tal manera que sea posible recuperar versiones especificas más adelante.

  2. Cual es la diferencia entre git y github?

    Git es el sistema de control de versiones, es un software permite tener un control de cambios sobre el código fuente de una aplicación. Fue inventado por Linus Torwalds, el mismo ingeniero que comenzó el desarrollo del Kernell de Linux. GitHub es una página que ofrece un grupo de servicios que facilitan el uso de Git, como por ejemplo hosting de proyectos, facilidades de colaboración, reviews de código, perfiles personales, pull requests, issues, etc.

  3. Cómo creo una rama en git?

    $ git branch nuevaRama : permite crear la rama nuevaRama $ git checkout -b nuevaRama : permite crear la rama nuevaRama y moverte a la rama nuevaRama

  4. Cómo clono un repositorio?

    $ git clone https://github.com/YOUR-USERNAME/YOUR-REPOSITORY (SSH de GitHub)

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    Al hacer $ git pull los cambios que hubiese en el servidor remoto se bajarán y fusionarán automáticamente. Si en lugar de eso, queremos bajar los cambios y fusionarlos de manera manual, podemos usar el comando $ git fetch (lo crea en otra rama), despues podemos realizar un $ git merge

  6. Cómo puedo gener un tag?

    $ git tag -a v1.4 -m 'my version 1.4' La opción -m especifica el mensaje de la etiqueta, el cual es guardado junto con ella. Si no especificas el mensaje de una etiqueta anotada, Git abrirá el editor de texto para que lo escribas.

  7. Cómo hago una mezcla de ramas?

    podemos usar uno de los dos siguientes comandos : $ git rebase o el $ git merge Se debe tener en cuenta que el rebase unifica las ramas perdiendo el historial de los commit y el merge no.

  8. Cómo puedo moverme a un commit especifico?

    Para volver a un commit especifico, primero encuentre el hash del commit utilizando git log. Luego para ir al commit manteniendo los cambios: $git reset --soft 789abcd. Tambien se puede utilizar el comando $ git checkout 789abcd, donde se creara una rama para los cambios en adelante.

  9. Cómo puedo borrar una rama?

    $ git branch -d nombre_rama : muestra si existe algun error, por ejemplo cambios sin fusionar $ git branch -D nombre-rama : Fuerza la eliminación de la rama Para un repositorio remoto $ git push origin --delete nombre_de_la_ram
